# CSS-архитектура MarketPlace

Стили разбиты на модули, чтобы блоки **не ломали друг друга** при правках.

## Точка входа

В шаблоне подключается только:

```
static/css/main.css
```

Файл `style.css` оставлен для совместимости и просто импортирует `main.css`.

## Слои каскада (`@layer`)

Порядок от слабого к сильному (последний побеждает только внутри своего слоя):

| Слой | Файл | Что внутри |
|------|------|------------|
| `reset` | `reset.css` | box-sizing, сброс отступов |
| `tokens` | `tokens.css` | CSS-переменные `:root` (цвета, радиусы) |
| `layout` | `layout.css` | шапка, навигация, футер, уведомления в шапке |
| `components` | `components.css` | кнопки, формы, карточки каталога, фильтры, `.dash-stats` |
| `blocks` | `blocks.css` | страницы: товар, чат, корзина, профиль, заявки, кошелёк… |
| `responsive` | `responsive.css` | все `@media` — **только здесь** |

Изменение стилей корзины не затронет кнопки, если править `blocks.css`, а не `components.css`.

## Именование (BEM)

- **Компонент:** `.card`, `.card__title`, `.btn`, `.btn--outline`
- **Блок страницы:** `.public-profile__stats`, `.cart-footer`, `.looking-request`
- **Дашборд продавца:** `.dash-stats`, `.dash-stat` — отдельно от `.pp-stat` в публичном профиле

Не добавляйте новые глобальные классы вроде `.stat` / `.item` / `.title` — они конфликтуют между страницами.

## Куда класть новые стили

| Задача | Файл |
|--------|------|
| Новая кнопка, инпут, бейдж | `components.css` |
| Шапка, меню, футер | `layout.css` |
| Новая страница или виджет | `blocks.css` (префикс блока: `.my-page__…`) |
| Мобильная вёрстка | `responsive.css` |
| Цвета, отступы по всему сайту | `tokens.css` |

## Пересборка после правок

Если снова собрали всё в один файл (не рекомендуется):

```bash
python scripts/split_css.py
```

Скрипт читает монолит `style.css`, переименовывает `.stats` → `.dash-stats` и раскладывает по модулям.

## Правила, чтобы не «слетало»

1. **Один блок — один префикс** (`.cart-*`, `.chat-*`, `.public-profile__*`).
2. **Не стилизуйте голые теги** в `blocks.css` (`div`, `p`) — только классы.
3. **Медиазапросы** — только в `responsive.css`, не размазывать по файлам.
4. **Переменные** — менять в `tokens.css`, не хардкодить цвета в блоках.
5. После правок — **Ctrl+F5** и проверка каталога + корзина + профиль.
