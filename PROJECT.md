# MarketPlace — документация проекта

Краткое описание для разработчиков и заказчика.

## Что это

Веб-маркетплейс (аналог Avito / Wildberries по базовому функционалу): каталог, объявления, корзина, заказы, чат, заявки «Ищу», кошелёк, режим продавца/покупателя.

**Стек:** Django, PostgreSQL/SQLite, шаблоны Django, CSS + vanilla JS.

**Демо:** `marketplace-e0cl.onrender.com` (Render, free tier — возможен cold start).

## Быстрый старт

```bash
python -m venv .venv
.venv\Scripts\activate          # Windows
pip install -r requirements.txt
python manage.py migrate
python manage.py seed           # демо-данные
python manage.py runserver
```

Тестовые аккаунты после `seed`:

| Логин  | Пароль    | Роль        |
|--------|-----------|-------------|
| admin  | admin123  | Админ       |
| seller | seller123 | Продавец    |
| buyer  | buyer123  | Покупатель  |

## Структура приложений

| Приложение   | Назначение |
|--------------|------------|
| `catalog`    | Каталог, фильтры, заявки «Ищу», бренды, категории |
| `listings`   | Объявления, отзывы, галерея |
| `orders`     | Корзина (гость + пользователь), заказы, оплата |
| `accounts`   | Профиль, верификация, кошелёк, соц. вход |
| `chat`       | Сообщения, поддержка с ботом |
| `panel`      | Админ-панель продавца/модератора |

## Корзина

- **Гость:** корзина хранится в сессии (`Cart.session_key`).
- **После входа:** товары сливаются в корзину пользователя (`orders/cart_service.py`).
- **Добавление:** AJAX + тост внизу экрана (как WB).
- **Количество:** автосохранение, итог пересчитывается на клиенте.

## Режим продавца

Переключение в профиле. Требуется: имя, телефон, подтверждённый email (`accounts/views.py` → `_seller_switch_errors`).

## Адрес доставки

Поле `User.delivery_address` в личном кабинете. Подставляется при оформлении заказа и сохраняется после checkout.

## Заявки «Ищу»

Покупатель оставляет запрос → продавец откликается объявлением → покупатель видит отклик в «Мои заявки».

## Деплой

См. `render.yaml`, переменные окружения в настройках Render. После деплоя: `migrate`, `seed` (по необходимости), `collectstatic`.

## Известные ограничения (демо)

- Оплата картой/ЮKassa — заглушка.
- SMS/email-коды — в консоль/лог (не реальный провайдер).
- Полный рефакторинг CSS — в планах; стили в `static/css/style.css`.
