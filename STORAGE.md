# Хранение фото и файлов

Cloudinary **недоступен в РФ** — используйте один из вариантов ниже.

## Вариант 1 — Без настройки (уже работает)

Файлы сохраняются на диске Render в папку `media/`.

**Минус:** после «сна» сервера (15 мин без посещений) загруженные файлы могут пропасть.  
**Плюс:** ничего настраивать не нужно, для тестов достаточно.

---

## Вариант 2 — Яндекс Object Storage (рекомендуется для РФ)

Постоянное хранение, работает в России, есть бесплатный стартовый грант.

### Шаги

1. Зайдите на [console.yandex.cloud](https://console.yandex.cloud) → создайте аккаунт
2. **Object Storage** → создайте **бакет** (например `marketplace-media`)
3. Сделайте бакет **публичным на чтение** (или настройте ACL для объектов)
4. **Сервисные аккаунты** → создайте SA → роль `storage.editor` → создайте **статический ключ** (Access Key + Secret)
5. В **Render** → ваш сервис `marketplace` → **Environment** → добавьте:

| Переменная | Значение |
|------------|----------|
| `AWS_STORAGE_BUCKET_NAME` | `marketplace-media` (имя вашего бакета) |
| `AWS_ACCESS_KEY_ID` | ключ из Яндекс |
| `AWS_SECRET_ACCESS_KEY` | секрет из Яндекс |
| `AWS_S3_ENDPOINT_URL` | `https://storage.yandexcloud.net` |
| `AWS_S3_REGION_NAME` | `ru-central1` |

6. **Manual Deploy** → перезапустите сайт

После этого фото в чате, объявлениях и аватары сохраняются в Яндекс навсегда.

## Вариант 3 — Selectel / Timeweb / другой S3

Любое S3-совместимое хранилище. Те же переменные:

```
AWS_STORAGE_BUCKET_NAME=ваш-бакет
AWS_ACCESS_KEY_ID=...
AWS_SECRET_ACCESS_KEY=...
AWS_S3_ENDPOINT_URL=https://...  (endpoint провайдера)
AWS_S3_REGION_NAME=...
```

Опционально:
```
AWS_S3_CUSTOM_DOMAIN=cdn.example.com
```

---

## Локальная разработка

Без переменных окружения файлы пишутся в папку `media/` в проекте — как обычно.
